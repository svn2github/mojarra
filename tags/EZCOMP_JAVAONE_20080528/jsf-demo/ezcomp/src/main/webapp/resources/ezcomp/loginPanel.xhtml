The composite component facility allows, but does not require, the composite component author to declare the usage contract of the component, which is comprised of:

  • the allowable attributes

    These can be thought of as a list of name=value pairs. "name" is the name of the attribute. "value" is the type of the attribute: ValueExpression or MethodExpression.

  • the allowable attached objects

    These can be thought of as a list of name=value pairs. "name" is the name of the attached object target, which must be a child component inside of the PDL file for the component. "value" is the kind of tag that represents the attached object. For example, "valueChangeListener" or "lengthValidator".

    • listeners

    • converters

    • validators

  • The facets supported by this component.

  • The allowable child components.

It is possible for the composite component author to declare the component-type of the top level component. If not declared, a script implemented component is looked for, with the same name as the composite component PDL file. If no script implemented component can be found, the default composite component impl is used.

Here is an example of a component contract that might be found in a component PDL file. The contract itself, and any of the content within it, are completely optional.



<ui:componentContract componentType="foo.loginPanel">

  <f:attribute name="usernameLabel" value="ValueExpression" 
               description="#{resource['this:usernameLabelDescription']}" />
  <f:attribute name="pinLabel" value="ValueExpression" />
  <f:attribute name="model" value="ValueExpression" />

  <f:valueChangeListener for="username" />
  <f:valueChangeListener for="password" />
  <f:actionListener for="loginButton" />
  <f:validateLength for="username" />
  <f:convertNumber for="pin" />

  <f:facet name="productName"/>

  <h:commandButton id="cancelButton" />

</ui:componentContract>
